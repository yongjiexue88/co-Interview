Use commits to record snapshots, branches to try features, and commands like commit, push, and pull to share work.</p><h3 class="text-xl font-bold mb-6 text-white">2. How does Git work?</h3><p class="mb-6 text-gray-300 leading-relaxed"><a href="https://git-scm.com/book/en/v2/Git-Internals-Git-Objects#what-is-a-blob" target="_self" class="text-white underline decoration-gray-400 underline-offset-2 hover:decoration-gray-600">Git records snapshots</a> of your project over time. You edit files in your working directory, stage chosen changes to the index, then commit a snapshot. Branches let you work in parallel, and merges or rebases combine work from different lines of development.</p><h3 class="text-xl font-bold mb-6 text-white">3. What is git add?</h3><p class="mb-6 text-gray-300 leading-relaxed">git add stages changes for the next commit. Example: git add file.txt stages that file. You can stage many files or parts of a file before committing the snapshot.</p><h3 class="text-xl font-bold mb-6 text-white">4. What is git push?</h3><p class="mb-6 text-gray-300 leading-relaxed">Git push uploads committed changes from your local branch to a remote like origin on GitHub. Example: git push origin main sends your local main branch to the remote main branch.</p><h3 class="text-xl font-bold mb-6 text-white">5. What is git status?</h3><p class="mb-6 text-gray-300 leading-relaxed">Git status shows which <a href="https://git-scm.com/book/en/v2/Git-Basics-Recording-Changes-to-the-Repository#_viewing_the_status" target="_self" class="text-white underline decoration-gray-400 underline-offset-2 hover:decoration-gray-600">files are modified</a>, which are staged, and which are untracked. What is a commit in Git?</h3><p class="mb-6 text-gray-300 leading-relaxed">A commit is a recorded snapshot of staged changes with a unique id and a message that explains why the change was made. What is branching in Git?</h3><p class="mb-6 text-gray-300 leading-relaxed">Branching creates a separate line of development so you can build features or fixes safely. What is a conflict in Git?</h3><p class="mb-6 text-gray-300 leading-relaxed">A conflict occurs when different changes affect the same lines, or when a file is deleted in one branch and modified in another. Git stops the merge and marks the conflict inside the file, allowing you to edit it. Then, you can use Git add and commit to resolve the file.</p><h3 class="text-xl font-bold mb-6 text-white">9. What is merging in Git?</h3><p class="mb-6 text-gray-300 leading-relaxed">Merging brings changes from one branch into another and produces a merge commit if needed.
