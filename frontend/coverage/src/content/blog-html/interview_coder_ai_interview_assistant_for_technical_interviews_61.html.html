What Is an EC2 Instance, and How Does It Operate?</h3><p class="mb-6 text-gray-300 leading-relaxed">An EC2 instance is a virtual machine you launch from an AMI. You pick an AMI, choose an instance type for CPU, memory, storage, and networking, and AWS runs it on the physical host. You manage the OS and applications unless you use managed compute options like ECS Fargate.</p><h3 class="text-xl font-bold mb-6 text-white">4. How Does S3 Differ from EBS, and When Do You Use Each?</h3><p class="mb-6 text-gray-300 leading-relaxed">S3 is object storage accessed by API or HTTP, ideal for static assets, backups, and data lakes. EBS is block storage attached to EC2, suitable for OS disks and databases that need low-latency IOPS. S3 is globally durable and zone-independent; EBS volumes are tied to a single Availability Zone.</p><h3 class="text-xl font-bold mb-6 text-white">5. How Does AWS Auto Scaling Work?</h3><p class="mb-6 text-gray-300 leading-relaxed">Auto Scaling adds or removes EC2 instances (or scales services like ECS or DynamoDB) based on policies, CloudWatch metrics, or scheduled actions. It keeps capacity aligned with demand, lowers cost during low traffic, and maintains availability when traffic spikes.</p><h3 class="text-xl font-bold mb-6 text-white">6. What Is the AWS Free Tier, and Which Services Are Included?</h3><p class="mb-6 text-gray-300 leading-relaxed">The Free Tier gives limited free usage for new accounts for 12 months and some services with always-free limits. What Are Key Pairs in AWS, and How Do They Secure Instances?</h3><p class="mb-6 text-gray-300 leading-relaxed">A key pair is an RSA or ED25519 public/private key pair. AWS stores the public key on the instance; you use the private key to SSH into Linux instances or decrypt Windows passwords. Key pairs prove identity and replace password-based access for EC2.</p><h3 class="text-xl font-bold mb-6 text-white">8. What Is Elastic Load Balancing, and What Does It Do?</h3><p class="mb-6 text-gray-300 leading-relaxed">Elastic Load Balancing distributes incoming traffic across targets, EC2, containers, or IP addresses, across multiple Availability Zones. What Types of Load Balancers Does AWS Offer, and When to Use Each?</h3><p class="mb-6 text-gray-300 leading-relaxed">Use Application Load Balancer for HTTP/HTTPS and content-based routing, Network Load Balancer for extreme performance and static IPs over TCP/UDP/TLS, and Gateway Load Balancer to insert third-party virtual appliances like firewalls. What Is Amazon RDS, and Which Database Engines Does It Support?</h3><p class="mb-6 text-gray-300 leading-relaxed">RDS is a managed relational database service that automates backups, patching, and failover. What Does AWS Identity and Access Management Do?</h3><p class="mb-6 text-gray-300 leading-relaxed">IAM creates and manages users, groups, roles, and fine-grained policies to grant or deny AWS API access. Use least privilege, roles for services, and MFA for sensitive access. What Is Amazon VPC, and How Does It Secure Resources?</h3><p class="mb-6 text-gray-300 leading-relaxed">VPC is a logically isolated virtual network. Use private subnets for databases and public subnets for internet-facing resources to reduce exposure.</p><h3 class="text-xl font-bold mb-6 text-white">14. What Is the Default Subnet Limit in a VPC?</h3><p class="mb-6 text-gray-300 leading-relaxed">A default soft limit is 200 subnets per VPC. What Is Amazon Route 53, and How Do You Use It?</h3><p class="mb-6 text-gray-300 leading-relaxed">Route 53 is a DNS and domain registration service that routes users to endpoints based on latency, geolocation, or health checks. How Does AWS Help with Backups and Disaster Recovery?</h3><p class="mb-6 text-gray-300 leading-relaxed">Use point-in-time RDS snapshots, S3 versioning and replication, AWS Backup for centralized policies, and cross-region replication for resilience. What Is Elastic Beanstalk, and How Does It Simplify Deployment?</h3><p class="mb-6 text-gray-300 leading-relaxed">Elastic Beanstalk deploys and manages applications by provisioning EC2, load balancers, Auto Scaling, and monitoring. Why Use AWS Organizations?</h3><p class="mb-6 text-gray-300 leading-relaxed">Organizations centralize billing, enable service control policies, and group accounts into OUs for consistent guardrails. Combine it with AWS Control Tower or Landing Zone to enforce security baselines and accelerate secure account provisioning.</p><h3 class="text-xl font-bold mb-6 text-white">19. What Is an AMI, and Why Use It?</h3><p class="mb-6 text-gray-300 leading-relaxed">An AMI is an image that includes an OS, software, and launch permissions. Save a configured instance as an AMI to launch identical servers quickly. Use encrypted AMIs and share AMIs across accounts or bake AMIs with tools like Packer for reproducible builds.</p><h3 class="text-xl font-bold mb-6 text-white">20. How Do Regions and Availability Zones Relate?</h3><p class="mb-6 text-gray-300 leading-relaxed">Regions are distinct geographic areas. What Is the Largest Object Size You Can Store in S3?</h3><p class="mb-6 text-gray-300 leading-relaxed">S3 supports single objects up to 5 terabytes. For large uploads, use a multipart upload to improve reliability and parallelism.</p><h3 class="text-xl font-bold mb-6 text-white">22. Reiterate the Differences Between S3 and EBS and Their Best Use Cases</h3><p class="mb-6 text-gray-300 leading-relaxed">S3 is ideal for immutable objects, backups, and static content with high durability and global access. EBS serves as block storage for running systems and databases that need consistent, low-latency IOPS. How Does Lambda Work, and What Are Typical Use Cases?</h3><p class="mb-6 text-gray-300 leading-relaxed">Lambda runs functions in response to events with automatic scaling and per-invocation billing. Lambda Fundamentals: How Serverless Execution Works</h3><p class="mb-6 text-gray-300 leading-relaxed">AWS Lambda runs code in response to events without managing servers, billing you only for execution time and resources used. Blue/Green Strategies: Minimize Risk with Dual Environments</h3><p class="mb-6 text-gray-300 leading-relaxed">Blue-green deployments hold two full environments and switch traffic to the new version once validated.
