What is Selenium?</h3><p class="mb-6 text-gray-300 leading-relaxed">Selenium is an open-source <a href="https://www.selenium.dev/about/" target="_self" class="text-white underline decoration-gray-400 underline-offset-2 hover:decoration-gray-600">tool for automating web browsers</a>. It lets you write scripts that mimic user actions, clicks, form fills, navigation, across browsers and operating systems.</p><p class="mb-6 text-gray-300 leading-relaxed">What to mention:</p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Works with Java, Python, C#, JavaScript, and more</li><li class="text-gray-300">Supports ChromeDriver, GeckoDriver, etc.</li><li class="text-gray-300">Includes IDE, WebDriver, and Grid (RC is outdated)</li></ul><h3 class="text-xl font-bold mb-6 text-white">2. What are the components of the Selenium suite?</h3><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Selenium IDE: Simple recorder for test prototypes</li><li class="text-gray-300">Selenium WebDriver: Core API for coded tests</li><li class="text-gray-300">Selenium Grid: Runs tests in parallel across environments</li><li class="text-gray-300">Selenium RC: Deprecated, don’t use it</li></ul><h3 class="text-xl font-bold mb-6 text-white">3. Why is Selenium so popular for automation?</h3><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Free and open-source</li><li class="text-gray-300">Supports multiple languages</li><li class="text-gray-300">Works on all major browsers and OSes</li><li class="text-gray-300">Integrates with TestNG, JUnit, Jenkins, and Grid</li></ul><h3 class="text-xl font-bold mb-6 text-white">4. What is test automation?</h3><p class="mb-6 text-gray-300 leading-relaxed">Test automation = writing scripts that run checks without human effort.</p><p class="mb-6 text-gray-300 leading-relaxed">Connect it to:</p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Faster feedback cycles</li><li class="text-gray-300">Consistent regression testing</li><li class="text-gray-300">Higher coverage in Agile/CI workflows</li></ul><h3 class="text-xl font-bold mb-6 text-white">5. What are the benefits of automation testing?</h3><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Saves time on repetitive tests</li><li class="text-gray-300">Catches bugs earlier</li><li class="text-gray-300">Reduces human error</li><li class="text-gray-300">Runs across browsers/versions</li></ul><h3 class="text-xl font-bold mb-6 text-white">6. What is Selenese?</h3><p class="mb-6 text-gray-300 leading-relaxed">The language used in Selenium IDE.</p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Actions: click, type, select</li><li class="text-gray-300">Accessors: storeText, storeTitle</li><li class="text-gray-300">Assertions: assertText, verifyElementPresent</li></ul><h3 class="text-xl font-bold mb-6 text-white">7. Selenium 2 vs Selenium 3?</h3><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Selenium 2 introduced WebDriver</li><li class="text-gray-300">Selenium 3 removed RC entirely, and WebDriver became the standard</li></ul><h3 class="text-xl font-bold mb-6 text-white">8. What types of testing does Selenium support?</h3><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Functional testing</li><li class="text-gray-300">Regression testing</li><li class="text-gray-300">End-to-end UI flows</li></ul><p class="mb-6 text-gray-300 leading-relaxed">Not suited for API or desktop/mobile apps.</p><h3 class="text-xl font-bold mb-6 text-white">9. What are the common TestNG annotations?</h3><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">@BeforeSuite, @AfterSuite</li><li class="text-gray-300">@BeforeTest, @AfterTest</li><li class="text-gray-300">@BeforeClass, @AfterClass</li><li class="text-gray-300">@BeforeMethod, @AfterMethod</li></ul><h3 class="text-xl font-bold mb-6 text-white">10. How do you locate web elements?</h3><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">By.id, By.name, By.className, By.tagName</li><li class="text-gray-300">By.cssSelector → fast and reliable</li><li class="text-gray-300">By.xpath → powerful but verbose</li><li class="text-gray-300">By.linkText, By.partialLinkText</li></ul><p class="mb-6 text-gray-300 leading-relaxed">Tip: Prefer IDs and CSS first. What are the different types of waits?</h3><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Implicit Wait: Sets a global timeout</li><li class="text-gray-300">Explicit Wait: Waits for a condition (e.g., element visible)</li><li class="text-gray-300">Fluent Wait: Custom polling, timeouts, ignored exceptions</li></ul><p class="mb-6 text-gray-300 leading-relaxed">WebDriverWait wait = new WebDriverWait(driver, Duration.ofSeconds(10));</p><p class="mb-6 text-gray-300 leading-relaxed">wait.until(ExpectedConditions.visibilityOfElementLocated(By.id("myId")));</p><h3 class="text-xl font-bold mb-6 text-white">12. Why does Selenium dominate the market?</h3><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Open-source</li><li class="text-gray-300">Works across OS, browser, and language</li><li class="text-gray-300">Integrates with CI/CD tools</li><li class="text-gray-300">Large community support</li></ul><h3 class="text-xl font-bold mb-6 text-white">19. Why use Selenium for web testing?</h3><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Cost-effective</li><li class="text-gray-300">Easy integration into pipelines</li><li class="text-gray-300">Runs parallel with Grid</li><li class="text-gray-300">Works with most frameworks</li></ul><h3 class="text-xl font-bold mb-6 text-white">20.
