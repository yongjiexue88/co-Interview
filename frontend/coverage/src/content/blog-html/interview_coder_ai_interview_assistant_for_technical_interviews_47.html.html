What is a Firewall?</h3><p class="mb-6 text-gray-300 leading-relaxed">A firewall is a hardware or software device that monitors incoming and outgoing traffic and allows, blocks, or drops packets based on configured security rules. Firewalls enforce access policies, isolate segments, and reduce exposure to unauthorized connections.</p><h3 class="text-xl font-bold mb-6 text-white">5. What is a VPN?</h3><p class="mb-6 text-gray-300 leading-relaxed">VPN stands for Virtual Private Network. A VPN creates an encrypted tunnel over an insecure network like the Internet so a remote user or site can securely access a private network. It uses tunneling protocols and encryption to protect data in transit and to extend private network access across public infrastructure.</p><h3 class="text-xl font-bold mb-6 text-white">6. How Does Email Work?</h3><p class="mb-6 text-gray-300 leading-relaxed">When a sender sends an email, the client hands it to an SMTP server. <a href="https://mailtrap.io/blog/smtp/?" target="_self" class="text-white underline decoration-gray-400 underline-offset-2 hover:decoration-gray-600">SMTP routes messages</a>, potentially through multiple mail servers, and consults DNS to find the recipient’s mail server. The recipient’s server stores the message until the recipient retrieves it using POP or IMAP. If servers cannot connect, messages may queue and be delivered later or returned as undeliverable if delivery fails for too long.</p><h3 class="text-xl font-bold mb-6 text-white">8. Active attacks threaten integrity and availability, often producing detectable changes.</li><li class="text-gray-300"><strong class="font-bold text-white">Passive Cyber Attack:</strong> The attacker observes or copies message content without altering it. Techniques include phone pretexts, phishing, and using public data like birthdays to guess credentials. Social engineering exploits trust and human routines.</p><h3 class="text-xl font-bold mb-6 text-white">10. Define Encryption and Decryption?</h3><p class="mb-6 text-gray-300 leading-relaxed">Encryption transforms plaintext into ciphertext using an algorithm and key so unauthorized parties cannot read it. Decryption reverses that process using the correct key to restore the original plaintext. Encryption protects confidentiality during storage and transit.</p><h3 class="text-xl font-bold mb-6 text-white">12. What is the Difference Between Plaintext and Cleartext?</h3><p class="mb-6 text-gray-300 leading-relaxed">Plaintext is data that has not been encrypted. Cleartext refers to information transmitted or stored without encryption and not intended to be encrypted. In both cases, the content is readable without decryption.</p><h3 class="text-xl font-bold mb-6 text-white">13. Modes like ECB and CBC describe how blocks chain together to <a href="https://www.techtarget.com/searchsecurity/definition/block-cipher?" target="_self" class="text-white underline decoration-gray-400 underline-offset-2 hover:decoration-gray-600">handle longer messages</a> and to add randomness and integrity properties.</p><h3 class="text-xl font-bold mb-6 text-white">14. What is the CIA Triangle?</h3><p class="mb-6 text-gray-300 leading-relaxed">The CIA Triad stands for Confidentiality, Integrity, and Availability. These three goals guide information security policies: keep data private, keep it accurate and unaltered, and keep services available when needed.</p><h3 class="text-xl font-bold mb-6 text-white">15. What is the Three-Way Handshake?</h3><p class="mb-6 text-gray-300 leading-relaxed">TCP uses a three-way handshake to establish a full-duplex reliable connection. The client sends SYN, the server replies SYN plus ACK, and the client sends ACK. Types and simple methods include:</p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Division Method</li><li class="text-gray-300">Mid Square Method</li><li class="text-gray-300">Folding Method</li><li class="text-gray-300">Multiplication Method</li></ul><p class="mb-6 text-gray-300 leading-relaxed">In cryptographic practice, algorithms like the SHA family and bcrypt are standard for integrity and password storage.</p><h3 class="text-xl font-bold mb-6 text-white">18. What Do You Mean by Two-Factor Authentication?</h3><p class="mb-6 text-gray-300 leading-relaxed">Two-factor authentication requires two independent methods to verify identity, typically something you know (password) plus something you have (token or phone) or something you are (biometrics). Consequences include:</p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Account compromise</li><li class="text-gray-300">Privilege escalation</li><li class="text-gray-300">Account deletion</li></ul><p class="mb-6 text-gray-300 leading-relaxed">Prevent XSS by filtering input as it arrives, encoding user data on output for the correct context (HTML, JavaScript, URL, CSS), sending strict response headers like Content-Type and X-Content-Type-Options, and using a Content Security Policy as a last line of defense.</p><h3 class="text-xl font-bold mb-6 text-white">20. This low-tech method can lead to privacy invasion and unauthorized access if people do not shield inputs.</p><h3 class="text-xl font-bold mb-6 text-white">21. What is the Difference Between Hashing and Encryption?</h3><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300"><strong class="font-bold text-white">Hashing:</strong> Converts data to a short fixed value used for indexing, integrity checks, and comparisons. Hashing is one-way; you cannot recover the original data from the hash. Hash outputs are fixed length, and collisions can occur.</li><li class="text-gray-300"><strong class="font-bold text-white">Encryption:</strong> Encodes data so authorized users with the key can reverse the process and recover the original plaintext. Encrypted output size varies with input length and the right key and algorithm allow decryption.</li></ul><h3 class="text-xl font-bold mb-6 text-white">22. With this approach, even if an attacker later obtains a private key, they cannot decrypt previously captured sessions that used ephemeral keys.</p><h3 class="text-xl font-bold mb-6 text-white">27. ARP Spoofing: Poisoning local network caches to redirect traffic through the attacker’s machine. </li><li class="text-gray-300"><strong class="font-bold text-white">Email Spoofing:</strong> Faking sender addresses and headers to phish or impersonate organizations.</li></ul><h3 class="text-xl font-bold mb-6 text-white">31. How Can It Be Prevented?</h3><p class="mb-6 text-gray-300 leading-relaxed">XSS stands for <a href="https://developer.mozilla.org/en-US/docs/Web/Security/Attacks/XSS?" target="_self" class="text-white underline decoration-gray-400 underline-offset-2 hover:decoration-gray-600">Cross-Site Scripting</a>. It allows attackers to inject scripts into pages viewed by other users, bypassing the same-origin protections. Prevent XSS by validating input, encoding output for the correct context, using safe response headers like Content-Type, and applying a Content Security Policy to limit what scripts can run.</p><h3 class="text-xl font-bold mb-6 text-white">34. What Are the Types of Cybersecurity?</h3><p class="mb-6 text-gray-300 leading-relaxed">Cybersecurity divides into sub-domains:</p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300"><strong class="font-bold text-white">Network security:</strong> Protects networks and traffic from intruders using firewalls and monitoring.</li><li class="text-gray-300"><strong class="font-bold text-white">Application security:</strong> Keeps software safe through secure coding, patching, and testing.</li><li class="text-gray-300"><strong class="font-bold text-white">Data security:</strong> Ensures data confidentiality and integrity while stored and in transit. Identity management: Controls who has access and what they can do with that access. </li><li class="text-gray-300"><strong class="font-bold text-white">Operational security:</strong> Processes and decisions to protect data handling and storage. Mobile security: Protects data on phones, tablets, and portable devices. </li><li class="text-gray-300"><strong class="font-bold text-white">Cloud security:</strong> Protects data and services hosted on cloud providers like AWS, Azure, and Google Cloud.</li></ul><h3 class="text-xl font-bold mb-6 text-white">36. Organizations and researchers use honeypots to study exploit methods and improve detection.</p><h3 class="text-xl font-bold mb-6 text-white">37.
