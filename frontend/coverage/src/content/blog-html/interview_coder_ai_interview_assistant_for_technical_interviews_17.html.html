They’re trying to determine if your past work aligns with what Intuit ships and if you can articulate yourself like a teammate, not a Wikipedia page.</p><p class="mb-6 text-gray-300 leading-relaxed">Sometimes the hiring manager jumps in. What Key Things Would You Look For When Reviewing Code?</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Rationale: They’re not just testing if it works; they care if someone else can read it after you’re gone.</li><li class="text-gray-300">Answer: Check for correctness, edge cases, naming, clean flow, short functions, minimal comments, and clear tradeoffs.</li></ul><h4 class="text-lg font-bold mb-6 text-white">2. Write Code To Check If A String Is A Palindrome</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Rationale: Tests string ops and attention to edge handling.</li><li class="text-gray-300">Answer: Two pointers inward, skip junk if needed, compare case-insensitively, O(n) time, O(1) space.</li></ul><h4 class="text-lg font-bold mb-6 text-white">3. Print All Permutations Of A String, Both Iteratively And Recursively</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Rationale: They want to see if you understand recursion vs control flow.</li><li class="text-gray-300">Answer: Use <a href="https://www.researchgate.net/figure/Recursive-swap-procedure_fig3_220060932" target="_self" class="text-white underline decoration-gray-400 underline-offset-2 hover:decoration-gray-600">recursive swaps</a> or Heap’s algorithm for iterative, mention O(n! * n), explain when to use each.</li></ul><h4 class="text-lg font-bold mb-6 text-white">4. Check If Two Strings Are Anagrams</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Rationale: Sorting tradeoffs and counting logic.</li><li class="text-gray-300">Answer: Sort and compare (O(n log n)) or count via hashmap/array (O(n)), depending on alphabet.</li></ul><h4 class="text-lg font-bold mb-6 text-white">5. Count How Often A Character Appears In A String</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Rationale: Looks simple. Easy to mess up.</li><li class="text-gray-300">Answer: Loop and count or use built-in methods, handle null/empty strings.</li></ul><h4 class="text-lg font-bold mb-6 text-white">6. Why Use Char[] Over String For Passwords?</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Rationale: They want to know if you think about memory and security.</li><li class="text-gray-300">Answer: Strings are immutable and hang around; char[] can be wiped from memory.</li></ul><h4 class="text-lg font-bold mb-6 text-white">7. Implement Insertion Sort In Java</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Rationale: Basic sort logic, in-place, and stability.</li><li class="text-gray-300">Answer: Iterate and shift, insert in place, mention O(n^2), and that it’s stable.</li></ul><h4 class="text-lg font-bold mb-6 text-white">8. Given Strings A And B, Find End Indices In B Where Its Substrings Are Prefixes Of A.</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Rationale: Pattern matching + KMP.</li><li class="text-gray-300">Answer: Preprocess A with KMP, match through B, and record match points. O(|A| + |B|)</li></ul><h4 class="text-lg font-bold mb-6 text-white">9. Given Arrays A And B, Count Pairs Where GCD(a, b) ≠ 1</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Rationale: Number theory and complexity control.</li><li class="text-gray-300">Answer: Count via prime factors, frequency arrays, or inclusion-exclusion.</li></ul><h3 class="text-xl font-bold mb-6 text-white">Data Structures &amp; Small Programs</h3><h4 class="text-lg font-bold mb-6 text-white">10. Return A List Of Duplicate Words In A Sentence</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Rationale: Tests tokenization + hashmap usage.</li><li class="text-gray-300">Answer: Split + normalize, count in a hashmap, return those &gt;1.</li></ul><h4 class="text-lg font-bold mb-6 text-white">11. Delete All Nodes Of A Given Value In A Linked List</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Rationale: Pointer manipulation and dummy nodes.</li><li class="text-gray-300">Answer: Use dummy head, two pointers, unlink matches, and skip carefully.</li></ul><h4 class="text-lg font-bold mb-6 text-white">12. Remove Duplicates From An Array</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Rationale: Sets, sorting, or in-place filtering.</li><li class="text-gray-300">Answer: Use a hash set (O(n) space), or sort and dedupe in-place (O(n log n)).</li></ul><h4 class="text-lg font-bold mb-6 text-white">13. What Are Divide And Conquer Algorithms?</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Rationale: Pattern recognition + recursion.</li><li class="text-gray-300">Answer: Break → solve → merge. Consider mergesort or quicksort as examples. Mention the Master Theorem.</li></ul><h4 class="text-lg font-bold mb-6 text-white">14. Find A Pair In A Sorted Array Whose Sum Is Closest To A Given Number N</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Rationale: Two-pointer test.</li><li class="text-gray-300">Answer: Start from both ends, track the closest. O(n).</li></ul><h4 class="text-lg font-bold mb-6 text-white">15. Reverse An Array</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Rationale: Indexing and basic loops.</li><li class="text-gray-300">Answer: Swap ends inward, or print backward. O(n) time.</li></ul><h4 class="text-lg font-bold mb-6 text-white">16. Reverse A String, But Keep Words In Place</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Rationale: Tests step-by-step thinking.</li><li class="text-gray-300">Answer: Reverse the whole string, then reverse each word. Or split/join.</li></ul><h4 class="text-lg font-bold mb-6 text-white">17. Spiral Order Traversal Of A Binary Tree</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Rationale: Level-order + alternation.</li><li class="text-gray-300">Answer: Use two stacks or a deque to flip direction at each level. O(n).</li></ul><h3 class="text-xl font-bold mb-6 text-white">Behavioral &amp; Situational</h3><h4 class="text-lg font-bold mb-6 text-white">18. What Makes You Eligible For This Role?</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Rationale: Fit and relevance.</li><li class="text-gray-300">Answer: Show outcomes tied to product goals. Use real metrics.</li></ul><h4 class="text-lg font-bold mb-6 text-white">19. How Do You Handle Challenging Situations?</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Rationale: Structured problem-solving.</li><li class="text-gray-300">Answer: Stabilize → root cause → short fix → long fix → comms.</li></ul><h4 class="text-lg font-bold mb-6 text-white">24.
