Here’s what’s in it and what actually matters.</p><h4 class="text-lg font-bold mb-6 text-white">Selenium IDE</h4><p class="mb-6 text-gray-300 leading-relaxed">If you’re new, this is your training wheels. What Selenium Actually Tests (Without the Fluff)</h3><p class="mb-6 text-gray-300 leading-relaxed">Alright, let’s keep this real.</p><p class="mb-6 text-gray-300 leading-relaxed">Most people crack open Selenium and immediately get buried in docs and buzzwords. Selenium Annotations in TestNG (Or: The Part Everyone Skims)</h3><p class="mb-6 text-gray-300 leading-relaxed">If you’re using TestNG with Selenium, you’ve seen those @Before and @After annotations. If they look like magic spells, here’s the cheat sheet:</p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">@BeforeSuite → runs before everything</li><li class="text-gray-300">@AfterSuite → runs after everything</li><li class="text-gray-300">@BeforeTest / @AfterTest → for the test methods</li><li class="text-gray-300">@BeforeClass / @AfterClass → before and after the test class methods</li><li class="text-gray-300">@BeforeMethod / @AfterMethod → before and after each test method</li><li class="text-gray-300">@BeforeGroups / @AfterGroups → before and after test groups</li></ul><p class="mb-6 text-gray-300 leading-relaxed">These are all about setup and cleanup. Web Element Locators: AKA: How to Find Stuff on a Page</h3><p class="mb-6 text-gray-300 leading-relaxed">Finding stuff on a page is half the battle. Here are the ways Selenium helps you do it:</p><h4 class="text-lg font-bold mb-6 text-white">1. ID</h4><p class="mb-6 text-gray-300 leading-relaxed">Best-case scenario. Fast and unique.</p><p class="mb-6 text-gray-300 leading-relaxed">driver.find_element(By.ID, "loginBtn")</p><h4 class="text-lg font-bold mb-6 text-white">2. Name</h4><p class="mb-6 text-gray-300 leading-relaxed">Often used in forms. Not always unique.</p><p class="mb-6 text-gray-300 leading-relaxed">driver.find_element(By.NAME, "email")</p><h4 class="text-lg font-bold mb-6 text-white">4. Tag Name</h4><p class="mb-6 text-gray-300 leading-relaxed">Useful if you’re selecting by type — like all &lt;input&gt;s.</p><p class="mb-6 text-gray-300 leading-relaxed">driver.find_element(By.TAG_NAME, "input")</p><h4 class="text-lg font-bold mb-6 text-white">5. CSS Selector</h4><p class="mb-6 text-gray-300 leading-relaxed">Flexible, clean. XPath</h4><p class="mb-6 text-gray-300 leading-relaxed">The Swiss Army knife. Link Text / Partial Link Text</h4><p class="mb-6 text-gray-300 leading-relaxed">Works great for anchor tags.</p><p class="mb-6 text-gray-300 leading-relaxed">driver.find_element(By.LINK_TEXT, "Sign up")</p><p class="mb-6 text-gray-300 leading-relaxed">XPath is overkill 90% of the time. Use it only when you have no other choice.</p><h3 class="text-xl font-bold mb-6 text-white">11. Waits in Selenium: Because Nothing Loads on Time</h3><p class="mb-6 text-gray-300 leading-relaxed">Web apps aren’t instant. You click a thing, and it takes a second (or five). Selenium needs to chill and wait sometimes.</p><h4 class="text-lg font-bold mb-6 text-white">Types of Waits</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Implicit Wait: Sets a global wait time. Implicit waits feel like using a sledgehammer to swat a fly.</p><h3 class="text-xl font-bold mb-6 text-white">12. Browser Navigation Commands: Like the Back Button, But Code</h3><p class="mb-6 text-gray-300 leading-relaxed">If you’ve ever wanted to automate browser actions like clicking “back” or refreshing, here’s your toolbox:</p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">.to("url") → go to a page</li><li class="text-gray-300">.back() → go back</li><li class="text-gray-300">.forward() → go forward</li><li class="text-gray-300">.refresh() → reload the page</li></ul><p class="mb-6 text-gray-300 leading-relaxed">That’s it. You can freestyle.</p><p class="mb-6 text-gray-300 leading-relaxed">Not really. Scripts follow scripts.</p><p class="mb-6 text-gray-300 leading-relaxed">Frameworks Needed</p><p class="mb-6 text-gray-300 leading-relaxed">Nah. Just you.</p><p class="mb-6 text-gray-300 leading-relaxed">Yep. You’ll need stuff like TestNG, PyTest, etc.</p><p class="mb-6 text-gray-300 leading-relaxed">If you’re running one test, go manual. If you’re testing every commit like a real dev team, automate it. There’s no pride in clicking the same button 47 times.</p><h3 class="text-xl font-bold mb-6 text-white">15. Absolute vs. Relative XPath: Which One’s Smarter?</h3><p class="mb-6 text-gray-300 leading-relaxed">Ever gotten lost because Google Maps gave you step-by-step directions from your house instead of where you’re currently standing? That’s Absolute XPath.</p><h4 class="text-lg font-bold mb-6 text-white">Absolute XPath</h4><p class="mb-6 text-gray-300 leading-relaxed">Starts from the root of the page and gives you the entire road trip. You’ll need something else.</p><h4 class="text-lg font-bold mb-6 text-white">No Image Validation</h4><p class="mb-6 text-gray-300 leading-relaxed">It can check if an image exists, not if it looks right.</p><h4 class="text-lg font-bold mb-6 text-white">No Built-In Reports</h4><p class="mb-6 text-gray-300 leading-relaxed">Bring your own TestNG or JUnit setup.</p><h4 class="text-lg font-bold mb-6 text-white">You Need Code</h4><p class="mb-6 text-gray-300 leading-relaxed">If you’re allergic to code, this isn’t your tool.</p><h3 class="text-xl font-bold mb-6 text-white">17. Same-Origin Policy: AKA “Why Won’t My Script Work?”</h3><p class="mb-6 text-gray-300 leading-relaxed">Browsers are paranoid for good reason.
