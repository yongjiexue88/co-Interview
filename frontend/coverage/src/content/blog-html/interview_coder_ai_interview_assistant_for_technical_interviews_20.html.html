These are the actual types of questions I got when prepping for interviews at Meta, Amazon, and TikTok, and the answers I would give today.</p><p class="mb-6 text-gray-300 leading-relaxed">Let’s go.</p><h3 class="text-xl font-bold mb-6 text-white">1. Why Were Client-Side Frameworks Like Angular Introduced?</h3><p class="mb-6 text-gray-300 leading-relaxed">Because manually wiring everything together in JavaScript got ridiculous.</p><p class="mb-6 text-gray-300 leading-relaxed">Apps got bigger. Code got messier. <a href="https://www.researchgate.net/publication/317637131_8_things_about_jQuery_which_makes_it_a_powerful_tool_to_write_rich_client_side_scripting" target="_self" class="text-white underline decoration-gray-400 underline-offset-2 hover:decoration-gray-600">Everyone was nesting jQuery</a> inside jQuery. You’d change one thing and break five. Frameworks showed up with a rule structure to help us not shoot ourselves in the foot.</p><h4 class="text-lg font-bold mb-6 text-white">Why They Ask It</h4><p class="mb-6 text-gray-300 leading-relaxed">They want to see if you understand the “why” behind the tech, not just how to use it.</p><h4 class="text-lg font-bold mb-6 text-white">How To Answer</h4><p class="mb-6 text-gray-300 leading-relaxed">Talk about scaling. Say something like “As web apps got more complex, frameworks like Angular helped enforce a pattern components, separation of concerns, routing so teams didn’t burn time fixing spaghetti.”</p><h4 class="text-lg font-bold mb-6 text-white">Roy’s Tip</h4><p class="mb-6 text-gray-300 leading-relaxed">Use a Lego analogy. It’s dead simple. “Components are like Lego blocks. You don’t want to glue them together. You want to swap them in and out without the whole tower collapsing.”</p><h3 class="text-xl font-bold mb-6 text-white">2. How Does An Angular Application Work?</h3><p class="mb-6 text-gray-300 leading-relaxed">High-level? Angular finds main.ts, runs bootstrapModule(AppModule), and renders AppComponent into index.html. That’s your entry point.</p><p class="mb-6 text-gray-300 leading-relaxed">From there:</p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">angular.json tells the CLI where stuff is,</li><li class="text-gray-300">main.ts spins things up</li><li class="text-gray-300">AppModule bundles all components.</li><li class="text-gray-300">AppComponent hits the screen.</li></ul><h4 class="text-lg font-bold mb-6 text-white">Why They Ask It</h4><p class="mb-6 text-gray-300 leading-relaxed">They're checking if you know how Angular initializes. Debugging weird startup errors starts here.</p><h4 class="text-lg font-bold mb-6 text-white">How To Answer</h4><p class="mb-6 text-gray-300 leading-relaxed">List the files in order: angular.json, main.ts, app.module.ts, app.component.ts, and finally index.html.</p><h4 class="text-lg font-bold mb-6 text-white">Roy’s Tip</h4><p class="mb-6 text-gray-300 leading-relaxed">Show you’ve actually debugged something before. Mention fixing a bootstrap array issue or a missing declaration once. That shows real-world experience.</p><h3 class="text-xl font-bold mb-6 text-white">3. What Are Some Advantages Of Angular Over Other Frameworks?</h3><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Routing, HTTP, forms, and state are all baked in</li><li class="text-gray-300">Templates are HTML-based (no JSX mental overhead)</li><li class="text-gray-300">Google’s backing = long-term stability</li></ul><h4 class="text-lg font-bold mb-6 text-white">Why They Ask It</h4><p class="mb-6 text-gray-300 leading-relaxed">They want to know if you can evaluate tradeoffs. Not just parrot “Angular good.”</p><h4 class="text-lg font-bold mb-6 text-white">How To Answer</h4><p class="mb-6 text-gray-300 leading-relaxed">Mention built-in tools and declarative templates, and explain why that saves dev time. Keep it concrete.</p><h3 class="text-xl font-bold mb-6 text-white">4. Angular Vs React: What's The Difference?</h3><p class="mb-6 text-gray-300 leading-relaxed">Short version: Angular gives you everything. React makes you assemble it yourself.</p><p class="mb-6 text-gray-300 leading-relaxed"><strong class="font-bold text-white">Data Binding:</strong></p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Angular → Two-way</li><li class="text-gray-300">React → One-way</li></ul><p class="mb-6 text-gray-300 leading-relaxed"><strong class="font-bold text-white">Dependency Injection:</strong></p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Angular → Built-in</li><li class="text-gray-300">React → Third-party</li></ul><p class="mb-6 text-gray-300 leading-relaxed"><strong class="font-bold text-white">Language:</strong></p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Angular → TypeScript</li><li class="text-gray-300">React → JavaScript</li></ul><p class="mb-6 text-gray-300 leading-relaxed"><strong class="font-bold text-white">Mobile Support:</strong></p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Angular → Yes</li><li class="text-gray-300">React → Only with React Native</li></ul><p class="mb-6 text-gray-300 leading-relaxed"><strong class="font-bold text-white">Tooling:</strong></p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Angular → Opinionated, built-in</li><li class="text-gray-300">React → Bring Your Own (BYO) everything</li></ul><p class="mb-6 text-gray-300 leading-relaxed">Don’t take sides. Say: “React’s great if you want freedom. Angular’s great if you want structure.”</p><h3 class="text-xl font-bold mb-6 text-white">5. Angular vs AngularJS?</h3><p class="mb-6 text-gray-300 leading-relaxed">This one’s easy if you’ve touched both.</p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Architecture: Angular uses components. AngularJS used controllers.</li><li class="text-gray-300">Language: Angular = TypeScript. AngularJS = JavaScript.</li><li class="text-gray-300">Mobile support: Angular works on mobile. AngularJS doesn’t.</li><li class="text-gray-300">Structure: Angular handles bigger apps better.</li><li class="text-gray-300">Syntax: Angular uses [ ] and ( ) for binding. AngularJS used ng-*.</li></ul><h4 class="text-lg font-bold mb-6 text-white">Why They Ask It</h4><p class="mb-6 text-gray-300 leading-relaxed">You might migrate legacy code. They want to know if you can handle that.</p><h3 class="text-xl font-bold mb-6 text-white">6. How are Angular Expressions Different From JavaScript expressions?</h3><p class="mb-6 text-gray-300 leading-relaxed">Angular expressions:</p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Only access the local component</li><li class="text-gray-300">Can’t access the window</li><li class="text-gray-300">Are safer</li><li class="text-gray-300">Support pipes</li></ul><h4 class="text-lg font-bold mb-6 text-white">Why They Ask It</h4><p class="mb-6 text-gray-300 leading-relaxed">They’re checking if you get how Angular avoids runtime surprises.</p><h4 class="text-lg font-bold mb-6 text-white">Roy’s Tip</h4><p class="mb-6 text-gray-300 leading-relaxed">Show an example where JS would return undefined, but Angular just shows nothing. Makes it click.</p><h3 class="text-xl font-bold mb-6 text-white">7. What is a Single Page Application (SPA)?</h3><p class="mb-6 text-gray-300 leading-relaxed">It’s an app that loads once, then swaps content via JS and client-side routing. No full page reloads.</p><h4 class="text-lg font-bold mb-6 text-white">Why They Ask It</h4><p class="mb-6 text-gray-300 leading-relaxed">SPAs change how data loads, how <a href="https://searchengineland.com/guide/what-is-seo" target="_self" class="text-white underline decoration-gray-400 underline-offset-2 hover:decoration-gray-600">search engine optimization (SEO)</a> works, and how fast users see changes.</p><h4 class="text-lg font-bold mb-6 text-white">Roy’s Tip</h4><p class="mb-6 text-gray-300 leading-relaxed">Mention tradeoffs: great UX, but SEO can be trickier, and first load time might suck.</p><h3 class="text-xl font-bold mb-6 text-white">8. What Are Templates In Angular?</h3><p class="mb-6 text-gray-300 leading-relaxed">Templates = HTML with Angular sauce (like {{ }} for binding, *ngIf, etc.)</p><p class="mb-6 text-gray-300 leading-relaxed">They live:</p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Inline (in the decorator)</li><li class="text-gray-300">Or in an external .html file via templateUrl</li></ul><h4 class="text-lg font-bold mb-6 text-white">Why They Ask It</h4><p class="mb-6 text-gray-300 leading-relaxed">They want to see if you understand where the view lives and how logic connects to it.</p><h3 class="text-xl font-bold mb-6 text-white">9. What Are Directives?</h3><p class="mb-6 text-gray-300 leading-relaxed">Directives change how things appear or behave.</p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Component: Has a view. Every component is a directive.</li><li class="text-gray-300">Structural: Changes layout. Ex: *ngIf, *ngFor</li><li class="text-gray-300">Attribute: Changes appearance. You can write your own.</li></ul><p class="mb-6 text-gray-300 leading-relaxed">Roy’s Tip</p><p class="mb-6 text-gray-300 leading-relaxed">You will be asked about *ngIf. Know it cold.</p><h3 class="text-xl font-bold mb-6 text-white">10. Components, Modules, and Services: Explain</h3><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Components = views + logic</li><li class="text-gray-300">Modules = group things together (can be root or feature modules)</li><li class="text-gray-300">Services = reusable logic or data fetchers</li></ul><p class="mb-6 text-gray-300 leading-relaxed">CLI makes all of these easy:</p><p class="mb-6 text-gray-300 leading-relaxed">ng g c your-component</p><p class="mb-6 text-gray-300 leading-relaxed">ng g m your-module</p><p class="mb-6 text-gray-300 leading-relaxed">ng g s your-service</p><h4 class="text-lg font-bold mb-6 text-white">Roy’s Tip</h4><p class="mb-6 text-gray-300 leading-relaxed">Mention providedIn: 'root' for services to get global singleton behavior.</p><h3 class="text-xl font-bold mb-6 text-white">11. What Is Scope?</h3><p class="mb-6 text-gray-300 leading-relaxed">This is AngularJS-era stuff.</p><p class="mb-6 text-gray-300 leading-relaxed">The scope was how AngularJS handled data binding and expression evaluation. Angular (the new one) replaced that with components and inputs/outputs.</p><h4 class="text-lg font-bold mb-6 text-white">Roy’s Tip</h4><p class="mb-6 text-gray-300 leading-relaxed">If they’re asking this, they might still have legacy code. Mention that you're familiar but prefer component-based architecture.</p><h3 class="text-xl font-bold mb-6 text-white">12. What Is Data Binding In Angular?</h3><p class="mb-6 text-gray-300 leading-relaxed">It’s how the component and template talk.</p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">{{}} — Interpolation</li><li class="text-gray-300">[property] — Property binding</li><li class="text-gray-300">(event) — Event binding</li><li class="text-gray-300">[(ngModel)] — Two-way binding</li></ul><h4 class="text-lg font-bold mb-6 text-white">Roy’s Tip</h4><p class="mb-6 text-gray-300 leading-relaxed">Say which one you’d use in a real-world scenario. Ex: "For inputs, I use two-way binding via ngModel so I don’t have to manually sync the state."</p><h3 class="text-xl font-bold mb-6 text-white">13. What Is Two-Way Data Binding?</h3><p class="mb-6 text-gray-300 leading-relaxed">It's just combining property + event binding:</p><p class="mb-6 text-gray-300 leading-relaxed">&lt;input [(ngModel)]="username"&gt;</p><p class="mb-6 text-gray-300 leading-relaxed">Any change to the username updates the input, and vice versa.</p><p class="mb-6 text-gray-300 leading-relaxed">You’ll need to import FormsModule in your module to use ngModel.</p><h3 class="text-xl font-bold mb-6 text-white">14. What Are Decorators?</h3><p class="mb-6 text-gray-300 leading-relaxed">They’re metadata annotations with @ that tell Angular what your class is.</p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">@Component — this is a UI piece</li><li class="text-gray-300">@NgModule — this is a module</li><li class="text-gray-300">@Injectable — this is a service</li><li class="text-gray-300">@Input / @Output — data in/out of components</li></ul><h3 class="text-xl font-bold mb-6 text-white">15. What Are Annotations?</h3><p class="mb-6 text-gray-300 leading-relaxed">In Angular, decorators replaced older annotation syntax. They do similar things, such as attach metadata to classes.</p><p class="mb-6 text-gray-300 leading-relaxed">Most of the time, you’ll just be working with decorators now.</p><h3 class="text-xl font-bold mb-6 text-white">16. What Are Pure Pipes?</h3><p class="mb-6 text-gray-300 leading-relaxed">They’re called only when the input changes by reference. Great for performance.</p><p class="mb-6 text-gray-300 leading-relaxed">Use them to format static-ish stuff like dates or currency.</p><h3 class="text-xl font-bold mb-6 text-white">17. What Are Impure Pipes?</h3><p class="mb-6 text-gray-300 leading-relaxed">They run every change detection cycle, even if inputs don’t change.</p><p class="mb-6 text-gray-300 leading-relaxed">You mark them like this:</p><p class="mb-6 text-gray-300 leading-relaxed">@Pipe({ name: 'something', pure: false })</p><p class="mb-6 text-gray-300 leading-relaxed">Only use these if you absolutely need to (like transforming data from a live stream or clock).</p><h3 class="text-xl font-bold mb-6 text-white">18. What is PipeTransform?</h3><p class="mb-6 text-gray-300 leading-relaxed">It’s the interface your custom pipes implement.</p><p class="mb-6 text-gray-300 leading-relaxed">export class MyPipe implements PipeTransform {</p><p class="mb-6 text-gray-300 leading-relaxed">transform(value: any, ...args: any[]): any {</p><p class="mb-6 text-gray-300 leading-relaxed">// do stuff</p><p class="mb-6 text-gray-300 leading-relaxed">}</p><p class="mb-6 text-gray-300 leading-relaxed">}</p><p class="mb-6 text-gray-300 leading-relaxed"></p><p class="mb-6 text-gray-300 leading-relaxed">You’ll need to import it from @angular/core.</p><h3 class="text-xl font-bold mb-6 text-white">19. How Do You Pass Data From Parent To Child?</h3><p class="mb-6 text-gray-300 leading-relaxed">Use @Input in the child component.</p><p class="mb-6 text-gray-300 leading-relaxed">// parent.component.html</p><p class="mb-6 text-gray-300 leading-relaxed">&lt;app-child [message]="someValue"&gt;&lt;/app-child&gt;</p><p class="mb-6 text-gray-300 leading-relaxed"></p><p class="mb-6 text-gray-300 leading-relaxed">// child.component.ts</p><p class="mb-6 text-gray-300 leading-relaxed">@Input() message: string;</p><h3 class="text-xl font-bold mb-6 text-white">20. What’s the difference between AOT and JIT?</h3><h4 class="text-lg font-bold mb-6 text-white">Real Talk</h4><p class="mb-6 text-gray-300 leading-relaxed">This question isn’t about definitions. It’s about whether you’ve touched <a href="https://www.ibm.com/think/topics/ci-cd-pipeline" target="_self" class="text-white underline decoration-gray-400 underline-offset-2 hover:decoration-gray-600">CI pipelines</a> and know why bundle size matters.</p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">AOT compiles templates at build time → faster startup, smaller runtime, earlier error detection.</li><li class="text-gray-300">JIT compiles in the browser at runtime → faster iteration during dev, slower startup.</li></ul><h4 class="text-lg font-bold mb-6 text-white">When I Use What</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Local dev = JIT for speed</li><li class="text-gray-300">CI &amp; prod = AOT for safety and performance ng build --aot is your friend.</li></ul><h3 class="text-xl font-bold mb-6 text-white">2. What Does @Component Actually Do?</h3><p class="mb-6 text-gray-300 leading-relaxed">It wires your class to the DOM.</p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Adds metadata like selector, templateUrl, and styleUrls</li><li class="text-gray-300">Angular uses that to figure out how to render your stuff.</li></ul><p class="mb-6 text-gray-300 leading-relaxed">Think of it as the config file that tells Angular, “This class belongs on this part of the page, styled like this, using this <a href="https://www.researchgate.net/publication/327861224_Introduction_to_HTML_Hyper_Text_Markup_Language_-_A_Review_Paper" target="_self" class="text-white underline decoration-gray-400 underline-offset-2 hover:decoration-gray-600">HyperText Markup Language (HTML)</a>.”</p><h3 class="text-xl font-bold mb-6 text-white">3. Promises vs Observables?</h3><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Promise = 1 value, not cancelable, eager</li><li class="text-gray-300">Observable = many values, cancelable, lazy</li></ul><p class="mb-6 text-gray-300 leading-relaxed">When I’m dealing with UI events, retry logic, or streams → Observable.</p><p class="mb-6 text-gray-300 leading-relaxed">When I just need to fetch() something and move on → Promise.</p><p class="mb-6 text-gray-300 leading-relaxed">Bonus: unsubscribe() saves memory.
