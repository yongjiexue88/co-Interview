Make the largest possible number from an array of non-negative integers, using a custom string comparator.</h3><p class="mb-6 text-gray-300 leading-relaxed">Given array of non negative integers, arrange them to form the largest possible concatenated number. Example: [1, 61, 9, 0] → "96110".</p><h4 class="text-lg font-bold mb-6 text-white">Core idea and algorithm</h4><p class="mb-6 text-gray-300 leading-relaxed">Treat numbers as strings. For two strings a and b compare a+b and b+a. Place a before b if a+b &gt; b+a. Sort the array with that comparator and join the results. Handle the all zero case by returning "0" when the highest element is "0".</p><h4 class="text-lg font-bold mb-6 text-white">Complexity and edge cases</h4><p class="mb-6 text-gray-300 leading-relaxed">Sorting cost O(n log n) string compare cost up to O(k) where k is average length of numbers, so effective O(n log n · k). Watch out for leading zeros, very long strings, and stable comparator implementation in languages with strict comparator contracts.</p><h4 class="text-lg font-bold mb-6 text-white">Coderpad tips</h4><p class="mb-6 text-gray-300 leading-relaxed">On Coderpad, write the comparator first and run a few examples interactively. Print intermediate ordering when debugging comparator logic. Ask the interviewer about expected output format and constraints.</p><h3 class="text-xl font-bold mb-6 text-white">2. Find all pairs that sum to a target in an array with positive and negative numbers, using hashing and two pointers.</h3><p class="mb-6 text-gray-300 leading-relaxed">Given an array with positive and negative numbers and a target sum, find all pairs whose sum equals target. Example: A=[1,2,3,4,5], target=4 → pair (1,3).</p><h4 class="text-lg font-bold mb-6 text-white">Common approaches</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Brute force O(n^2): check each pair.</li><li class="text-gray-300">Hash set O(n): while iterating, check if target - current exists. Add current into set after the check to avoid using the same element twice.</li><li class="text-gray-300">Two-pointer O(n log n) if you first sort the array. Use left and right pointers to move inward.</li></ul><h4 class="text-lg font-bold mb-6 text-white">Complexity and edge cases</h4><p class="mb-6 text-gray-300 leading-relaxed">Hash approach runs in O(n) time and O(n) space. Watch duplicate elements and whether you should return indices or values. Decide whether pairs must be unique and whether the array is allowed to contain repeating elements.</p><h4 class="text-lg font-bold mb-6 text-white">Coderpad tips</h4><p class="mb-6 text-gray-300 leading-relaxed">Ask whether to return indices or values. Write test cases with duplicates and negatives. Voice your plan then implement. Run small test cases on the pad.</p><h3 class="text-xl font-bold mb-6 text-white">3. Word break: check string segmentation using a given dictionary, recursion with memoization, or DP.</h3><p class="mb-6 text-gray-300 leading-relaxed">Given a dictionary and an input string, determine whether the string can be segmented into a sequence of dictionary words. Example: dict = {"bat", "cricket"}, input "batcricketcricket" → true.</p><h4 class="text-lg font-bold mb-6 text-white">Algorithm</h4><p class="mb-6 text-gray-300 leading-relaxed">Dynamic programming or recursion with memoization. Let dp[i] be true if s[i:] can be segmented. For i from len(s) down to 0, try all words that match starting at i. If any leads to dp[j] true, set dp[i] true. Recursion without memo can explode, so memoize by index.</p><h4 class="text-lg font-bold mb-6 text-white">Complexity and edge cases</h4><p class="mb-6 text-gray-300 leading-relaxed">Time O(n · m) where n is length of string and m average branching due to dictionary lookup. Use a trie or limit checking to word lengths in dictionary to speed checks. Watch empty string, repeated reuse of a word, and large dictionaries.</p><h4 class="text-lg font-bold mb-6 text-white">Coderpad tips</h4><p class="mb-6 text-gray-300 leading-relaxed">Start with a clear recurrence on the pad. Add a couple of asserts for strings that are true and false. Explain complexity and memory cost during the interview.</p><h3 class="text-xl font-bold mb-6 text-white">4. Find the missing number in an array from range 0 to n , using the sum or XOR trick.</h3><p class="mb-6 text-gray-300 leading-relaxed">Given n distinct numbers from range 0 to n with one missing, find the missing number. Example: {5,2,3,1,0} → missing 4.</p><h4 class="text-lg font-bold mb-6 text-white">Algorithms</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300"><strong class="font-bold text-white">Sum formula: </strong>expected = n(n+1)/2, missing = expected - sum(array). Watch overflow for large n use 64 bit types.</li><li class="text-gray-300"><strong class="font-bold text-white">XOR trick: </strong>xor all numbers from 0..n then xor with all array elements; result is missing number. This avoids overflow.</li></ul><h4 class="text-lg font-bold mb-6 text-white">Complexity and edge cases</h4><p class="mb-6 text-gray-300 leading-relaxed">Both solutions run in O(n) time and O(1) extra space. Handle unsorted input, duplicated inputs if any, and data type limits.</p><h4 class="text-lg font-bold mb-6 text-white">Coderpad tips</h4><p class="mb-6 text-gray-300 leading-relaxed">Ask about constraints and integer size. On the pad, implement both and run a test that shows the XOR method avoids overflow.</p><h3 class="text-xl font-bold mb-6 text-white">5. Print Fibonacci sequence using recursion, recursion basics, and tail recursion…</h3><p class="mb-6 text-gray-300 leading-relaxed">Print Fibonacci numbers where F(0)=0, F(1)=1, F(n)=F(n-1)+F(n-2). Example: for n=10 output 55 for F(10).</p><h4 class="text-lg font-bold mb-6 text-white">Approaches</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Naive recursion directly follows the recurrence. That yields exponential time O(2^n).</li><li class="text-gray-300">Use memoization or an iterative loop to run in O(n) time.</li><li class="text-gray-300">Use tail recursion or matrix exponentiation for faster results for huge n.</li></ul><h4 class="text-lg font-bold mb-6 text-white">Complexity and edge cases</h4><p class="mb-6 text-gray-300 leading-relaxed">Avoid naive recursion in interviews unless you then optimize. Memoized recursion or iterative loop gives O(n) time and O(n) or O(1) space, respectively. Watch negative n and huge n where overflow occurs.</p><h4 class="text-lg font-bold mb-6 text-white">Coderpad tips</h4><p class="mb-6 text-gray-300 leading-relaxed">Start with the recursive version to communicate correctness, then add memoization or change to an iterative version. Run a few small n before scaling.</p><h3 class="text-xl font-bold mb-6 text-white">6. Check the primality of a number using efficient trial division.</h3><p class="mb-6 text-gray-300 leading-relaxed">Determine if n is prime. Clarify that 0 and 1 are not prime. A prime has exactly two positive divisors: 1 and itself.</p><h4 class="text-lg font-bold mb-6 text-white">Algorithm</h4><p class="mb-6 text-gray-300 leading-relaxed">Check small cases n &lt; 2 false. Test divisibility by two separately, then test odd divisors from 3 up to sqrt(n). Stop early when a divisor is found. For significant inputs, use probabilistic tests like Miller-Rabin.</p><h4 class="text-lg font-bold mb-6 text-white">Complexity and edge cases</h4><p class="mb-6 text-gray-300 leading-relaxed">Trial division runs in O(sqrt(n)). Use 64-bit math to avoid overflow when squaring, for big integers, use fast primality tests. Consider negative numbers, 0, 1, and even numbers.</p><h4 class="text-lg font-bold mb-6 text-white">Coderpad tips</h4><p class="mb-6 text-gray-300 leading-relaxed">Ask about input size. On the pad, implement trial division and add a note that Miller-Rabin is required for huge integers. Run basic tests, including small primes and composites.</p><h3 class="text-xl font-bold mb-6 text-white">7. Longest common prefix among several strings compared to the min string or vertical scan.</h3><p class="mb-6 text-gray-300 leading-relaxed">Given an array of strings, find the longest common prefix. Remove every instance of a character from a string in-place or using built-in methods…..</h3><p class="mb-6 text-gray-300 leading-relaxed">Remove all occurrences of character c from string s. Watch Unicode and case sensitivity. If memory matters, prefer in-place mutation for mutable buffers.</p><h4 class="text-lg font-bold mb-6 text-white">Coderpad tips</h4><p class="mb-6 text-gray-300 leading-relaxed">On Coderpad, show both a simple one-liner and an in-place routine. Run tests that include repeated targets and characters not present.</p><h3 class="text-xl font-bold mb-6 text-white">9. Find the longest line in a text file with Python file streaming and key functions.</h3><p class="mb-6 text-gray-300 leading-relaxed">Given a text file, return the longest line. Example one liner: print(max(open('test.txt'), key=len)).</p><h4 class="text-lg font-bold mb-6 text-white">Better practice</h4><p class="mb-6 text-gray-300 leading-relaxed">Use with open to close file cleanly: with open('test.txt') as f: print(max(f, key=len).rstrip('\n')). For large files, iterate and track the max length to avoid keeping all content in memory.</p><h4 class="text-lg font-bold mb-6 text-white">Complexity and edge cases</h4><p class="mb-6 text-gray-300 leading-relaxed">Time O(L) where L is the total file size. Handle trailing newlines and empty files.</p><h4 class="text-lg font-bold mb-6 text-white">Coderpad tips</h4><p class="mb-6 text-gray-300 leading-relaxed">If the interviewer allows a scripting solution, show the one-liner, then improve it to handle empty file and file path parameters.</p><h3 class="text-xl font-bold mb-6 text-white">10. Implement Bubble sort, explain tradeoffs and stable swap logic.</h3><p class="mb-6 text-gray-300 leading-relaxed">Implement bubble sort to arrange numbers in ascending or descending order. It swaps adjacent elements when they are out of order.</p><h4 class="text-lg font-bold mb-6 text-white">Algorithm and code sketch</h4><p class="mb-6 text-gray-300 leading-relaxed">Classic double loop:for i in range(n-1): for j in range(n-i-1): if a[j] &gt; a[j+1]: swap a[j], a[j+1]Optimized version stops early if no swaps occur in a pass.</p><h4 class="text-lg font-bold mb-6 text-white">Complexity and edge cases</h4><p class="mb-6 text-gray-300 leading-relaxed">Worst and average time O(n^2). Only use in interviews to show understanding of elementary sorts and prove correctness.</p><h4 class="text-lg font-bold mb-6 text-white">Coderpad tips</h4><p class="mb-6 text-gray-300 leading-relaxed">Explain why bubble sort is rarely used in production, and then implement the optimized early exit. Swap two numbers without a third variable, arithmetic, or XOR.</h3><p class="mb-6 text-gray-300 leading-relaxed">Swap x and y without a temporary variable. Example x=52, y=67.</p><h4 class="text-lg font-bold mb-6 text-white">Approaches</h4><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Arithmetic: x = x + y; y = x - y; x = x - y. Watch overflow risk.</li><li class="text-gray-300">XOR bitwise: x = x ^ y; y = x ^ y; x = x ^ y. This avoids overflow but needs integer types.</li></ul><h4 class="text-lg font-bold mb-6 text-white">Complexity and edge cases</h4><p class="mb-6 text-gray-300 leading-relaxed">O(1) time O(1) space. Avoid when variables refer to the memory location because XOR/arith methods can zero out values. Use with caution in languages with limited integer width.</p><h4 class="text-lg font-bold mb-6 text-white">Coderpad tips</h4><p class="mb-6 text-gray-300 leading-relaxed">Ask whether integers are safe from overflow. On the pad, try both methods and include a test where x and y are equal.</p><h3 class="text-xl font-bold mb-6 text-white">12. Find all non-overlapping subarrays that sum to a target prefix sum, using a hash map and greedy two-pointer.</h3><p class="mb-6 text-gray-300 leading-relaxed">Return all non-overlapping subarrays that sum to the target. Use prefix sums with a hash map to detect sums quickly and ensure subarrays do not overlap.</p><h4 class="text-lg font-bold mb-6 text-white">Algorithm</h4><p class="mb-6 text-gray-300 leading-relaxed">Compute prefix sum pref. As you scan, check if pref-target exists in the hash map mapping sum to the last end index. When you find a candidate subarray (start = map[pref - target] + 1, end = current index), accept it only if start &gt; lastAcceptedEnd to enforce non-overlapping.<br><br>Update lastAcceptedEnd to end. This gives a greedy O(n) pass. For listing all possible non-overlapping choices, you can store indices and use two pointer, like scanning and skipping the used range.</p><h4 class="text-lg font-bold mb-6 text-white">Complexity and edge cases</h4><p class="mb-6 text-gray-300 leading-relaxed">Single pass O(n) time and O(n) space. Handle negative numbers, zeros, and repeated sums. If you need the maximum count of non-overlapping subarrays, use greedy earliest end acceptance.</p><h4 class="text-lg font-bold mb-6 text-white">Coderpad tips</h4><p class="mb-6 text-gray-300 leading-relaxed">Clarify whether non-repeating means non-overlapping. On the pad, write a test with overlapping candidates and show the greedy choice of the earliest finishing segment.</p><h3 class="text-xl font-bold mb-6 text-white">13. Convert Excel column number to title base 26 without zero.</h3><p class="mb-6 text-gray-300 leading-relaxed">Convert a positive integer to an Excel column string. Example 27 → "AA".</p><h4 class="text-lg font-bold mb-6 text-white">Algorithm</h4><p class="mb-6 text-gray-300 leading-relaxed">Use 1-based digit mechanics. While n &gt; 0: decrement n by 1, letter = chr(n % 26 + ord('A')), prepend letter, then n //= 26. This accounts for the missing zero digit in this base.</p><h4 class="text-lg font-bold mb-6 text-white">Complexity and edge cases</h4><p class="mb-6 text-gray-300 leading-relaxed">Loop runs O(log26(n)) steps. Handle huge n. Confirm that input starts at 1, not 0.</p><h4 class="text-lg font-bold mb-6 text-white">Coderpad tips</h4><p class="mb-6 text-gray-300 leading-relaxed">Write the loop and run examples 1→A, 26→Z, 27→AA, 52→AZ, 703→AAA.</p><h3 class="text-xl font-bold mb-6 text-white">14.
