Engineer Interview (15 Q&amp;A)</h2><div class="mb-8"><span data-media="image">[IMAGE 800x600]</span></div><h3 class="text-xl font-bold mb-6 text-white">1. How Would You Select The Top Five Most Expensive Projects By Budget To Employee Count Ratio?</h3><p class="mb-6 text-gray-300 leading-relaxed">Think of this as a “do you understand joins or are you just winging it?” question. Count the distinct employees per project, watch out for garbage <a href="https://www.linkedin.com/posts/edwigesongong_how-do-you-handle-null-values-in-your-dataset-activity-7376587925859442688-5FzK" target="_self" class="text-white underline decoration-gray-400 underline-offset-2 hover:decoration-gray-600">data such as NULLs</a> or duplicate rows, compute the budget / employee_count, and sort the results. Nothing fancy, just clean SQL with a working brain behind it.</p><h4 class="text-lg font-bold mb-6 text-white">Why Would Interviewers Ask This?</h4><p class="mb-6 text-gray-300 leading-relaxed">They’re checking whether you can reason through messy data without panicking. If you don’t think about duplicate join rows or the “divide by zero because a project has zero engineers” situation, they’ll see it immediately.</p><h4 class="text-lg font-bold mb-6 text-white">How Should You Answer?</h4><p class="mb-6 text-gray-300 leading-relaxed">Lay out your steps calmly:</p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Dedup employees first.</li><li class="text-gray-300">Compute employee counts.</li><li class="text-gray-300">Join the projects table.</li><li class="text-gray-300">Do the ratio with a safeguard.</li><li class="text-gray-300">Mention indexing if the dataset’s huge, not because you're showing off, but because it demonstrates respect for performance.</li></ul><h4 class="text-lg font-bold mb-6 text-white">Sample Query</h4><p class="mb-6 text-gray-300 leading-relaxed">WITH emp_counts AS (</p><p class="mb-6 text-gray-300 leading-relaxed">SELECT</p><p class="mb-6 text-gray-300 leading-relaxed">project_id,</p><p class="mb-6 text-gray-300 leading-relaxed">COUNT(DISTINCT employee_id) AS employee_count</p><p class="mb-6 text-gray-300 leading-relaxed">FROM employee_projects</p><p class="mb-6 text-gray-300 leading-relaxed">GROUP BY project_id</p><p class="mb-6 text-gray-300 leading-relaxed">)</p><p class="mb-6 text-gray-300 leading-relaxed">SELECT</p><p class="mb-6 text-gray-300 leading-relaxed">p.project_id,</p><p class="mb-6 text-gray-300 leading-relaxed">p.budget,</p><p class="mb-6 text-gray-300 leading-relaxed">ec.employee_count,</p><p class="mb-6 text-gray-300 leading-relaxed">CASE WHEN ec.employee_count &gt; 0 THEN p.budget::numeric / ec.employee_count ELSE NULL END AS budget_per_employee</p><p class="mb-6 text-gray-300 leading-relaxed">FROM projects p</p><p class="mb-6 text-gray-300 leading-relaxed">LEFT JOIN emp_counts ec ON p.project_id = ec.project_id</p><p class="mb-6 text-gray-300 leading-relaxed">WHERE p.budget IS NOT NULL</p><p class="mb-6 text-gray-300 leading-relaxed">ORDER BY budget_per_employee DESC NULLS LAST</p><p class="mb-6 text-gray-300 leading-relaxed">LIMIT 5;</p><h3 class="text-xl font-bold mb-6 text-white">2. How Would You Generate A Monthly Report Of User Transactions And Total Order Amount For The Year 2020?</h3><p class="mb-6 text-gray-300 leading-relaxed">This is the bread-and-butter reporting question. Group by month, count users, count transactions, and sum totals. If some months have zero activity, fake the calendar and include them.</p><h4 class="text-lg font-bold mb-6 text-white">Why Ask This?</h4><p class="mb-6 text-gray-300 leading-relaxed">They want to see if you understand dates beyond “yeah, I know what January is.” Also, to check if you can make a clean report that doesn’t magically skip months because the table had no rows.</p><h4 class="text-lg font-bold mb-6 text-white">How Should You Answer?</h4><p class="mb-6 text-gray-300 leading-relaxed">Filter first (fast), then bucket by month, aggregate, and finally join to a generated month series so that dead months are visible instead of disappearing, as if they were deleted out of shame.</p><h4 class="text-lg font-bold mb-6 text-white">Sample Query</h4><p class="mb-6 text-gray-300 leading-relaxed">WITH months AS (</p><p class="mb-6 text-gray-300 leading-relaxed">SELECT generate_series('2020-01-01'::date, '2020-12-01'::date, interval '1 month') AS month_start</p><p class="mb-6 text-gray-300 leading-relaxed">),</p><p class="mb-6 text-gray-300 leading-relaxed">tx AS (</p><p class="mb-6 text-gray-300 leading-relaxed">SELECT</p><p class="mb-6 text-gray-300 leading-relaxed">date_trunc('month', t.created_at)::date AS month_start,</p><p class="mb-6 text-gray-300 leading-relaxed">COUNT(*) AS transaction_count,</p><p class="mb-6 text-gray-300 leading-relaxed">COUNT(DISTINCT t.user_id) AS user_count,</p><p class="mb-6 text-gray-300 leading-relaxed">SUM(t.total_amount) AS total_amount</p><p class="mb-6 text-gray-300 leading-relaxed">FROM transactions t</p><p class="mb-6 text-gray-300 leading-relaxed">WHERE t.created_at &gt;= '2020-01-01' AND t.created_at &lt; '2021-01-01'</p><p class="mb-6 text-gray-300 leading-relaxed">GROUP BY 1</p><p class="mb-6 text-gray-300 leading-relaxed">)</p><p class="mb-6 text-gray-300 leading-relaxed">SELECT</p><p class="mb-6 text-gray-300 leading-relaxed">to_char(m.month_start, 'YYYY-MM') AS month,</p><p class="mb-6 text-gray-300 leading-relaxed">COALESCE(tx.user_count, 0) AS user_count,</p><p class="mb-6 text-gray-300 leading-relaxed">COALESCE(tx.transaction_count, 0) AS transaction_count,</p><p class="mb-6 text-gray-300 leading-relaxed">COALESCE(tx.total_amount, 0) AS total_amount</p><p class="mb-6 text-gray-300 leading-relaxed">FROM months m</p><p class="mb-6 text-gray-300 leading-relaxed">LEFT JOIN tx ON tx.month_start = m.month_start</p><p class="mb-6 text-gray-300 leading-relaxed">ORDER BY m.month_start;</p><h3 class="text-xl font-bold mb-6 text-white">3. How Would You Identify Customers Who Placed More Than Three Transactions In Both 2019 And 2020?</h3><p class="mb-6 text-gray-300 leading-relaxed"><a href="https://www.researchgate.net/figure/Variable-recognition-of-conditional-aggregation_tbl2_357711669" target="_self" class="text-white underline decoration-gray-400 underline-offset-2 hover:decoration-gray-600">Conditional aggregation</a>. That’s it. Don’t overthink it.</p><h4 class="text-lg font-bold mb-6 text-white">Why Ask This?</h4><p class="mb-6 text-gray-300 leading-relaxed">This filters out the people who memorize syntax but never reason about <em class="italic text-gray-300">what</em> they’re counting. It’s the SQL version of: “Are you awake at the wheel?”</p><h4 class="text-lg font-bold mb-6 text-white">How Should You Answer?</h4><p class="mb-6 text-gray-300 leading-relaxed">Use SUM(CASE WHEN...) in one pass.Or say you’d intersect two sets.Either way, stay clear and don’t do weird nested subqueries that make the engineer beside you wonder if you’re okay.</p><h4 class="text-lg font-bold mb-6 text-white">Sample Query</h4><p class="mb-6 text-gray-300 leading-relaxed">SELECT user_id</p><p class="mb-6 text-gray-300 leading-relaxed">FROM transactions</p><p class="mb-6 text-gray-300 leading-relaxed">GROUP BY user_id</p><p class="mb-6 text-gray-300 leading-relaxed">HAVING</p><p class="mb-6 text-gray-300 leading-relaxed">SUM(CASE WHEN EXTRACT(YEAR FROM created_at) = 2019 THEN 1 ELSE 0 END) &gt; 3</p><p class="mb-6 text-gray-300 leading-relaxed">AND</p><p class="mb-6 text-gray-300 leading-relaxed">SUM(CASE WHEN EXTRACT(YEAR FROM created_at) = 2020 THEN 1 ELSE 0 END) &gt; 3;</p><h3 class="text-xl font-bold mb-6 text-white">4. How Would You Write A Function To Rotate An Array 90 Degrees Clockwise?</h3><p class="mb-6 text-gray-300 leading-relaxed">Transpose + reverse rows. That’s the whole trick. Matrix problems only feel intimidating if you treat them like dark magic.</p><h4 class="text-lg font-bold mb-6 text-white">Why Ask This?</h4><p class="mb-6 text-gray-300 leading-relaxed">They want to know if you get how data moves inside a matrix, not whether you can memorize language quirks.</p><h4 class="text-lg font-bold mb-6 text-white">How Should You Answer?</h4><p class="mb-6 text-gray-300 leading-relaxed">Explain the algorithm as if you’ve actually done it before, not as if you read it 5 minutes ago on Stack Overflow.</p><h3 class="text-xl font-bold mb-6 text-white">5. How Would You Generate A Transposed Matrix And Estimate Parameters For Linear Regression?</h3><p class="mb-6 text-gray-300 leading-relaxed">You build X, build Xᵀ, form the standard equation, and solve using pseudo-inverse because the universe loves to hand you singular matrices at the worst time.</p><h4 class="text-lg font-bold mb-6 text-white">Why Ask This?</h4><p class="mb-6 text-gray-300 leading-relaxed">Disney wants engineers who can think in math, not just code levers. This question exposes whether you understand what linear regression actually <em class="italic text-gray-300">is</em>.</p><h4 class="text-lg font-bold mb-6 text-white">How Should You Answer?</h4><p class="mb-6 text-gray-300 leading-relaxed">Show the flow:</p><ul class="mb-6 space-y-2 pl-6"><li class="text-gray-300">Build X and y.</li><li class="text-gray-300">Add an intercept.</li><li class="text-gray-300">Compute Xᵀ.</li><li class="text-gray-300">Solve coefficients with pseudo-inverse.</li><li class="text-gray-300">Mention stability so they know you’re not blindly calling np.linalg.inv like it's 2012.</li></ul><h3 class="text-xl font-bold mb-6 text-white">6. Write reverse_at_position(Head, K) That Reverses From K To The End.</h3><p class="mb-6 text-gray-300 leading-relaxed">Classic pointer question. They’re checking if you can handle a linked list without creating a crime scene.</p><h4 class="text-lg font-bold mb-6 text-white">Why Ask This?</h4><p class="mb-6 text-gray-300 leading-relaxed">People freeze up on pointer problems.
